# Active Plans

Top-level `.plans/` is intentionally small.

Keep only:

- current workflow truth that still matches the shipped editor
- currently active proposals that still align with the present UX/codebase

Current precedence:

1. tests and implementation
2. `AGENTS.md`
3. `.repo-memory/`
4. top-level `.plans/`
5. `.plans/archive/`

If a plan or mockup no longer matches the shipped product closely enough to guide new work, move it to `.plans/archive/` instead of leaving it at the top level.
