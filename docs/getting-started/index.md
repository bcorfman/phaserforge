# Getting Started

Use these pages in order if you are new to PhaserForge and want the intended cloud-first path from first login to a published demo.

## Recommended Path

1. [Set Up Your Cloud Account](./cloud-account-setup)
2. [Build the Pattern Demo](./pattern-demo)
3. [Publish to GitHub Pages](./publish-to-github-pages)

## What These Pages Assume

- You can run PhaserForge locally.
- You want to use PhaserForge the normal way: signed in, with a cloud account, and with GitHub available for publish.
- You may need reference links back to editor workflow names and controls while you work.

Offline-only use exists, but it should be treated as an exception for debugging, temporary connectivity issues, or narrowly local experiments, not as the regular first-user path.

Keep the [Editor Workflow Reference](../reference/editor-workflows) open if you want the exact names for controls, shortcuts, and fallback entry points.
